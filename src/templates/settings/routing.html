If an incoming request’s URL doesn’t match a page or a template, these “routes” are Blocks’ last resort before responding with a 404. A route basically says, “If the URL looks like this, then return that.”

Each route is composed of a pattern which Blocks compares the URL with, and the template to load if it’s a match. Patterns usually contain “vars” that could match a range of different things, and Blocks makes their matches available as template variables.
